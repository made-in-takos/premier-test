INTRODUCTION

Sass est une extension de CSS qui ajoute puissance et élégance au language CSS. Comme pour beaucoup de language de programmation, il vous permet d'utiliser des variables ou des règles imbriquées, mais aussi des mixins, des importations, des sélecteurs multiples, etc... Le tout, avec une syntaxe entièrement compatible CSS.
SASS vous aidera à garder de grandes feuilles de style bien organisées, en particulier avec l'aide de la bibliothèque de style Compass.

Caractéristiques
  • Entièrement compatible avec CSS.
  • Des extensions de langage telles que des variables, des fonctions ou encore des mixins.
  • De nombreuses fonctions utiles pour manipuler les couleurs et autres valeurs.
  • Des fonctions avancées comme les directives de contrôle pour les bibliothèques.
  • Des sorties formatées et personnalisables.

Syntaxe

Il existe deux syntaxes disponibles pour Sass. La première, connue sous le nom SCSS (Sassy CSS) sera utilisée tout au long de cette référence, c' est une extension de la syntaxe CSS.
Cela signifie que chaque feuille de style CSS est un fichier compilé depuis SCSS, ayant la même signification. En outre, SCSS comprend la plupart des hacks CSS et les syntaxes spécifiques, comme la syntaxe de filtre d'IE. Cette syntaxe est améliorée avec les fonctionnalités Sass décrites ci-dessous. Les fichiers utilisant cette syntaxe ont l'extension .scss.

La deuxième syntaxe est plus ancienne, connue sous le nom de syntaxe indentée (ou parfois seulement "Sass"), elle fournit une manière plus concise d'écrire du CSS, mais maoins claire au niveau de la lecture du code.

SASS utilise l'indentation plutôt que les crochets pour indiquer l'emboîtement des sélecteurs. De plus, il utilise une nouvelle ligne plutôt que les points-virgules pour séparer les propriétés. Si certaines personnes trouvent cela plus facile à lire et plus rapide à écrire que le SCSS, le système d'accolade et de point virgule (qui se retrouvant aussi dans le CSS) délimite mieux visuellement les différents blocs. Les fichiers utilisant la syntaxe SASS ont l'extension .sass.

Les fichiers peuvent être automatiquement convertis d'une syntaxe à l'autre à l'aide de l'outil de ligne de commande sass-convert: 


# Convertir Sass en SCSS
$ sass-convert style.sass style.scss

# Convertir SCSS en Sass
$ sass-convert style.scss style.sass

Utiliser Sass en lignes de commandes

Sass peut être utilisé de plusieurs façons différentes:
en tant qu'outil de lignes de commandes, en tant que module Ruby autonome et en tant que plugin pour n'importe quel framework, y compris Ruby on Rails et Merb. La première étape pour tout cela est d'installer Sass: 

gem install sass

Si vous utilisez Windows, vous devrez peut-être installer Ruby en premier (http://bkaprt.com/sass/5/). Pour savoir si Ruby est déja installé, tapez dans l'invite de commandes. 

 ruby --help

Pour exécuter Sass depuis la ligne de commande, il suffit d'utiliser 

sass input.scss output.css

Vous pouvez également indiquer à SASS de regarder le fichier et ainsi mettre à jour le CSS chaque fois que le fichier Sass change: 

sass --watch input.scss:output.css

Si vous avez un répertoire avec de nombreux fichiers Sass, vous pouvez également dire à Sass de regarder le répertoire entier: 

sass --watch app/sass:public/stylesheets

Utilisez sass --help pour obtenir toute la documentation.

Utiliser Sass avec scout-app

Si les lignes de commandes ne sont pas au centre de vos rêves les plus palpitants, il existe des applications de bureau qui peuvent faciliter la gestion et la conversion de vos fichiers SASS/SCSS. Dans cette référence, nous utiliserons scoot, (http://scout-app.io/) qui est entièrement gratuit et Open source, mais il en existe d'autres :

  • scout (que nous venons de citer) pour linux, windows et mac.
  • Fire.app (http://fireapp.kkbox.com) pour linux, windows et mac.
  • Codekit (https://codekitapp.com) uniquement pour mac et payant, bien que peux onéreux.
Configurer scout.

Une fois le téléchargement terminé, et l'installation effectuée, il vous faudra créer un nouveau projet pour ensuite lui indiquer le chemain d'entrée (SCSS) et de sortie (CSS) de vos fichiers.

Exemple
Input folder: 

/~/stylesheets/SCSS/

Output folder: 

/~/stylesheets/CSS/

Cliquez ensuite sur le bouton lecture de votre projet, et Scout démarrera le suivit des modifications apportés au fichier SCSS.
Il ne sera alors plus utile de manipuler le fichier CSS car sa compilation se fera automatiquement via l'application.
